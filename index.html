Node.js File System y Path

En Node.js, los módulos fs y path son fundamentales para trabajar con el sistema de archivos y manipular rutas de archivos y directorios, respectivamente.

fs (File System)

Este módulo proporciona una API para interactuar con el sistema de archivos del sistema operativo en el que se ejecuta Node.js. Te permite realizar operaciones como leer archivos, escribir archivos, crear directorios, eliminar archivos, entre otros.

Algunos métodos comunes de fs incluyen:


            fs.readFile(): Lee el contenido de un archivo.
            fs.writeFile(): Escribe datos en un archivo.
            fs.readdir(): Lee el contenido de un directorio.
            fs.mkdir(): Crea un nuevo directorio.
            fs.unlink(): Elimina un archivo.
            fs.stat(): Obtiene información sobre un archivo o directorio.

Ejemplo de uso:


            const fs = require("fs");

            fs.readFile("archivo.txt", "utf8", (err, data) => {
                if (err) throw err;
                console.log(data);
            });

path

Este módulo proporciona utilidades para trabajar con rutas de archivos y directorios. Esto es útil para manejar rutas de forma independiente del sistema operativo, ya que las rutas pueden ser diferentes en Windows, Linux, etc.

Algunos métodos comunes de path incluyen:


            path.join(): Une segmentos de ruta en una sola ruta.
            path.resolve(): Resuelve una secuencia de segmentos de ruta en una ruta absoluta.
            path.basename(): Devuelve el último segmento de la ruta.
            path.dirname(): Devuelve el directorio padre de la ruta.

Ejemplo de uso:


            const path = require("path");

            const fullPath = path.join(__dirname, "archivo.txt");
            console.log(fullPath);
